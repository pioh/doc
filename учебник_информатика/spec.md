# СПЕЦИФИКАЦИЯ ПРОЕКТА: Учебник по Информатике для Первокурсников

## Цель проекта

Создать полноценный учебник по информатике, охватывающий 43 темы из 9 разделов, который позволит первокурсникам полностью подготовиться к экзамену.

## Структура учебника

### Разделы и темы:

**Раздел 1: Понятие информации. Общая характеристика процессов сбора, передачи, обработки и накопления информации** (12 тем)

- Темы 1-12

**Раздел 2: Технические средства реализации информационных процессов** (3 темы)

- Темы 13-15

**Раздел 3: Программные средства реализации информационных процессов** (3 темы)

- Темы 16-18

**Раздел 4: Модели решения функциональных и вычислительных задач** (4 темы)

- Темы 19-22

**Раздел 5: Основы алгоритмизации** (2 темы)

- Темы 23-24

**Раздел 6: Языки программирования высокого уровня** (2 темы)

- Темы 25-26

**Раздел 7: Базы данных** (3 темы)

- Темы 27-29

**Раздел 8: Локальные и глобальные сети ЭВМ** (7 тем)

- Темы 30-36

**Раздел 9: Основы защиты информации и сведений составляющих государственную тайну** (7 тем)

- Темы 37-43

## Требования к содержанию

### Общие требования:

1. **Академический стиль**: Учебник должен быть написан понятным, но академическим языком
2. **Полнота изложения**: Каждая тема должна быть раскрыта полностью с примерами
3. **Структурированность**: Четкие заголовки, подзаголовки, списки, таблицы где необходимо
4. **Практичность**: Включать примеры, схемы (описание), задачи для самопроверки
5. **Связность**: Каждая глава должна логично вытекать из предыдущей и готовить к следующей

### Требования к каждой главе:

1. **Объем**: 3-8 страниц печатного текста (примерно 5000-15000 символов)
2. **Структура главы**:

   - Заголовок темы
   - Введение (актуальность, связь с другими темами)
   - Основная часть с подразделами
   - Примеры и иллюстрации (текстовое описание)
   - Ключевые термины и определения
   - Контрольные вопросы (3-5 вопросов)

3. **Стиль изложения**:
   - От простого к сложному
   - Использование аналогий для сложных понятий
   - Выделение ключевых терминов
   - Ссылки на другие главы где уместно

## Технический процесс создания

### Файловая структура:

```
учебник_информатика/
├── spec.md                    # Этот файл - спецификация
├── progress.md                # Трекер прогресса
├── table_of_contents.md       # Оглавление
├── README.md                  # Инструкции для агентов
├── agent_notes.md             # Заметки между агентами
├── context_summary.md         # Краткая сводка для контекста
├── build_script.py            # Скрипт сборки в DOCX
└── chapters/                  # Главы учебника
    ├── 01_01_арифметика_двоичная.md
    ├── 01_02_представление_данных_эвм.md
    ├── ...
    └── 09_07_юридические_основы_иб.md
```

### Именование файлов глав:

Формат: `РР_ТТ_краткое_название.md`

- РР - номер раздела (01-09)
- ТТ - номер темы в разделе (01-12)
- краткое_название - понятное описание латиницей/кириллицей

### Процесс работы агентов:

1. **Перед началом работы** агент ОБЯЗАН:

   - Прочитать spec.md (этот файл)
   - Прочитать progress.md (узнать, что уже сделано)
   - Прочитать context_summary.md (получить сводку предыдущего контекста)
   - Прочитать agent_notes.md (заметки от предыдущих агентов)

2. **Во время работы** агент:

   - Пишет ОДНУ главу за сессию
   - Создает или редактирует соответствующий файл в папке chapters/
   - Следует всем требованиям из этой спецификации

3. **После завершения** агент ОБЯЗАН:
   - Обновить progress.md (отметить выполненную тему)
   - Обновить context_summary.md (краткая сводка новой главы)
   - Обновить agent_notes.md (заметки для следующего агента)
   - Указать, какую главу писать следующей

### Формат записи в progress.md:

```
- [x] 01.01 Арифметика в двоичной системе счисления ✓
- [ ] 01.02 Представление данных в памяти ЭВМ
```

### Формат context_summary.md:

Краткие тезисы (2-3 предложения) по каждой написанной главе для поддержания контекста.

### Формат agent_notes.md:

Заметки от агента к агенту:

- Особенности стиля
- Какие термины уже введены
- Что важно учесть в следующих главах
- Предложения по улучшению

## Финальная сборка

После написания всех глав запускается скрипт `build_script.py`, который:

1. Собирает все главы в правильном порядке
2. Добавляет оглавление
3. Форматирует текст
4. Создает файл учебник_информатика.docx
5. Готовый файл можно печатать

## Критерии качества

Учебник считается готовым, если:

1. ✓ Все 43 темы раскрыты полностью
2. ✓ Каждая глава соответствует требованиям по структуре и объему
3. ✓ Материал изложен доступно для первокурсников
4. ✓ Есть примеры и контрольные вопросы
5. ✓ Логическая связность между главами
6. ✓ Итоговый файл DOCX успешно собирается

## Целевая аудитория

Первокурсники технических специальностей, изучающие информатику. Предполагаемый уровень:

- Базовые знания работы с компьютером
- Школьный курс информатики
- Минимальные знания математики (10-11 класс)

## Важные замечания

1. **Не писать несколько глав за раз** - это перегрузит контекст
2. **Всегда обновлять служебные файлы** - это критически важно для следующих агентов
3. **Поддерживать единый стиль** - читать предыдущие главы для согласованности
4. **Ориентироваться на экзамен** - давать именно ту информацию, которая нужна для сдачи
